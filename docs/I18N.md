# Internationalization Reference — SherpaCarta

## Site Implementation (Current)

SherpaCarta uses **client-side UI translation** in `index.html`:

| Code | Language | Hero/CTA | Charter full text |
|------|----------|----------|-------------------|
| `en` | English | ✅ | ✅ (source) |
| `zh` | Chinese | ✅ | Community |
| `es` | Spanish | ✅ | Community |
| `ar` | Arabic | ✅ (+ RTL) | Community |
| `fr` | French | ✅ | Community |

Language switcher: nav dropdown → `switchNavLang()` → `localStorage` key `sc_lang`.

**SEO:** `hreflang` tags point to `?lang=` query URLs; deep-link handling on load is planned (see `docs/SEO.md`).

## Give A Bit Shared System

This project can SHARE the Give A Bit translation system for future expansion.
Master translations live at: `~/projects/giveabit/src/i18n/`

Available in master: `en`, `es`, `fr`, `sw`, `zh`, `pt`, `hi`, `ar`

To add project-specific terms, edit the master files and add a `sherpacarta.` prefix key.

## Locale SEO Docs

Per-language keyword baselines: `docs/SEO-{es,fr,de,pt,zh,sw}.md`

## Translation Workflow (Recommended)

1. Contributor opens issue/PR with locale + articles translated
2. Review for legal nuance (not literal machine translation for binding language)
3. Merge charter JSON/block into `index.html` or future `charter/{lang}.json`
4. Update `docs/SEO-{lang}.md` meta recommendations
